# AgentMirror — terminal walkthrough

A 30-second tour of the whole value loop. Run it yourself (throwaway temp dir, nothing touched in your repo):

```bash
./demo/walkthrough.sh
```

It uses a global `agentmirror` if installed, otherwise the local build in this repo.

## What it shows

```text
### 1. The mess — two tools, two files, already contradicting each other
CLAUDE.md says tabs; .cursorrules says spaces. Which does the agent believe?

### 2. Consolidate into ONE canonical AGENTS.md
(in real life, `agentmirror init` imports an existing file for you)

### 3. Generate every tool's mirror from it
agentmirror sync
  ✓ CLAUDE.md (updated)
  ✓ .cursor/rules/agents.mdc (created)
  2 file(s) written from AGENTS.md.
--- generated CLAUDE.md (top) ---
<!-- Generated by agentmirror from AGENTS.md. Do not edit directly — edit AGENTS.md and run `agentmirror sync`. -->

# Project rules

### 4. Drift can't sneak back — someone hand-edits a mirror:
agentmirror check
  ✗ CLAUDE.md (drifted — run `agentmirror sync`)
  ✓ .cursor/rules/agents.mdc
  Problems found — see above.
exit code: 1  (non-zero = CI fails)

### 5. Re-sync fixes it
agentmirror check
  ✓ CLAUDE.md
  ✓ .cursor/rules/agents.mdc
  In sync.
exit code: 0

### 6. Stale rules get caught too — reference a file that doesn't exist:
agentmirror check
  ✓ CLAUDE.md
  ✓ .cursor/rules/agents.mdc
  ✗ path "src/deleted.ts" was not found in the repo (line 8)
  Problems found — see above.
exit code: 1

Done. One file to edit, every tool in sync, drift + stale rules caught in CI.
```
